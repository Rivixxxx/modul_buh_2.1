1. Общие принципы ведения банковских операций

Важно: Все банковские операции должны быть своевременно отражены в учете и соответствовать первичным документам.

Основные задачи

  • Своевременное создание и отправка платежных поручений
  • Контроль поступления средств от покупателей
  • Ведение взаиморасчетов с контрагентами
  • Информирование заинтересованных подразделений о движении средств
  • Подготовка отчетности по банковским операциям

Документооборот

Шаг 1: Получение первичных документов (счета, договоры, акты)
Шаг 2: Создание платежного поручения в 1С или банковской системе
Шаг 3: Согласование и подписание документов
Шаг 4: Отправка в банк для исполнения
Шаг 5: Получение и обработка банковских выписок

2. Создание платежных поручений

2.1 Создание платежных поручений в 1С

Шаг 1: Перейдите в раздел "Банк и касса" → "Платежные поручения"
Шаг 2: Нажмите кнопку "Создать" для создания нового документа
Шаг 3: Заполните обязательные реквизиты:
  • Плательщик (ваша организация)
  • Получатель (контрагент)
  • Сумма платежа
  • Назначение платежа
  • Банковские реквизиты получателя
Шаг 4: Проверьте корректность всех данных
Шаг 5: Проведите документ и установите статус "Подготовлено"
Внимание: Обязательно проверяйте корректность банковских реквизитов получателя перед отправкой платежа.

2.2 Работа с системами Клиент-Банк

Шаг 1: Войдите в систему Клиент-Банк вашего банка
Шаг 2: Выберите раздел "Платежи" или "Платежные поручения"
Шаг 3: Создайте новый платежный документ
Шаг 4: При создании платежа в банковской системе заполните:
  • Счет списания
  • Получатель и его реквизиты
  • Сумма и валюта
  • Назначение платежа
  • Очередность платежа
Шаг 5: Подпишите документ электронной подписью
Шаг 6: Отправьте в банк на исполнение

3. Интеграция и обмен данными с 1С

3.1 Настройка системы обмена с Клиент-Банком

Шаг 1: В журнале "Платежные поручения" нажмите "Отправить в банк"
Шаг 2: Перейдите в "Настройка" на вкладке "Отправка в банк"
Шаг 3: Укажите параметры:
  • Банковский счет организации
  • Программу клиент-банка
  • Файл выгрузки в банк (по умолчанию: 1c_to_kl)
  • Файл загрузки из банка (по умолчанию: kl_to_1c)
Шаг 4: Настройте реквизиты по умолчанию:
  • Группа для новых контрагентов
  • Статья доходов
  • Статья расходов
Шаг 5: Установите параметры выгрузки и загрузки:
  • Виды выгружаемых документов
  • Контроль корректности номера документа
  • Автоматическое создание новых элементов
  • Автоматическое проведение документов
Рекомендация: Используйте сервис 1С:ДиректБанк для более современного и удобного обмена данными с банком.

3.2 Выгрузка платежных документов

Шаг 1: Убедитесь, что у платежных поручений статус "Подготовлено" или "Согласовано"
Шаг 2: Откройте форму "Клиент-банк" из журнала платежных поручений
Шаг 3: На вкладке "Отправка в банк" отметьте документы для выгрузки
Шаг 4: Нажмите "Выгрузить" для создания файла обмена
Шаг 5: Загрузите созданный файл в программу Клиент-Банк
Шаг 6: Проверьте загруженные документы и отправьте их в банк
Безопасность: Если включен контроль безопасности, не закрывайте окно проверки до завершения загрузки в банк.

3.3 Загрузка банковских выписок

Шаг 1: Выгрузите банковские выписки из программы Клиент-Банк
Шаг 2: В 1С перейдите в журнал "Банковские выписки"
Шаг 3: Нажмите кнопку "Загрузить"
Шаг 4: Выберите файл выписки для загрузки
Шаг 5: Проверьте корректность загруженных данных:
  • Контрагенты (отмечены серым, если будут созданы автоматически)
  • Договоры (сверьте с назначением платежа)
  • Суммы и даты операций
Шаг 6: Отметьте выписки для загрузки и нажмите "Загрузить"

4. Разнесение банковских выписок

4.1 Типовые операции по банковским выпискам

Тип операции Дебет Кредит Примечание
Поступление от покупателя 51 62 Оплата за товары/услуги
Предоплата от покупателя 51 62 Авансовые платежи
Оплата поставщику 60 51 Погашение задолженности
Перечисление налогов 68 51 Платежи в бюджет
Социальные взносы 69 51 Взносы в фонды
Банковские комиссии 91.2 51 Расходы по операциям

4.2 Алгоритм разнесения выписок

Шаг 1: Откройте загруженную банковскую выписку
Шаг 2: Проверьте автоматически определенные данные:
  • Контрагент
  • Договор
  • Статья движения ДС
  • Аналитика
Шаг 3: Скорректируйте данные при необходимости:
  • Выберите правильный договор
  • Укажите корректную статью доходов/расходов
  • Проверьте НДС
Шаг 4: Для неопределенных операций создайте новые элементы справочников
Шаг 5: Проведите документ и проверьте правильность проводок
Важная информация: При автоматической загрузке система создает договор "Основной договор" для новых контрагентов.

4.3 Контроль правильности разнесения

Шаг 1: Сверьте остатки по счету 51 с банковскими остатками
Шаг 2: Проверьте правильность отражения в регистрах:
  • Взаиморасчеты с контрагентами
  • Расчеты по налогам и сборам
  • Движение денежных средств
Шаг 3: Используйте отчет "Анализ субконто" для контроля
Шаг 4: Исправьте выявленные ошибки через корректировочные документы

5. Создание реестра банковских операций для отдела продаж

Шаг 1: Определите период для формирования реестра (обычно месяц)
Шаг 2: Используйте стандартный отчет "Оборотно-сальдовая ведомость по счету 51"
Шаг 3: Настройте отбор по следующим критериям:
  • Период: выбранный месяц
  • Счет: 51 (Расчетные счета)
  • Субконто: Контрагенты
  • Группировка: по контрагентам и операциям
Шаг 4: Дополнительно создайте детализированный отчет:
  • Журнал документов "Поступление на расчетный счет"
  • Отбор по дате и типу операции
  • Группировка по контрагентам
Шаг 5: Оформите реестр в следующем формате:
№ Дата Контрагент Сумма поступления Назначение платежа Договор Примечание
1 01.01.2024 ООО "Покупатель 1" 100 000,00 Оплата по счету №123 Договор №456 Полная оплата
2 02.01.2024 ИП Иванов И.И. 50 000,00 Предоплата Договор №789 Аванс 50%
Шаг 6: Добавьте итоговые суммы:
  • Общая сумма поступлений
  • Количество операций
  • Распределение по видам платежей
Шаг 7: Передайте реестр в отдел продаж до 5 числа следующего месяца

6. Информирование отдела продаж

6.1 Ежедневное информирование о поступлениях

Шаг 1: Ежедневно до 10:00 формируйте сводку поступлений за предыдущий день
Шаг 2: Используйте отчет "Движение денежных средств" с отбором:
  • Период: вчерашний день
  • Тип операции: Поступление
  • Детализация по контрагентам
Шаг 3: Отправьте информацию по email с темой: "Поступления за [дата]"

6.2 Информирование об ошибочных платежах

Шаг 1: При обнаружении ошибочного платежа немедленно уведомите ОП
Шаг 2: Укажите в уведомлении:
  • Дату и сумму платежа
  • Плательщика
  • Назначение платежа
  • Причину ошибки
  • Предлагаемые действия
Шаг 3: Согласуйте с ОП порядок возврата или зачета платежа

6.3 Возврат задолженностей покупателям

Шаг 1: Получите от ОП заявление на возврат с указанием:
  • Основания для возврата
  • Суммы к возврату
  • Реквизиты для перечисления
Шаг 2: Проверьте правомочность возврата и наличие средств
Шаг 3: Создайте платежное поручение на возврат
Шаг 4: После исполнения платежа уведомите ОП о выполнении возврата
